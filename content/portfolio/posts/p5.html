Этот код позволяет парсить информацию о вакансиях на hh.ru
Он сканирует начальную страницу, находит вакансии, переходит по ссылкам на вакансии и собирает указанные данные

Импорт и настройки

Основными библиотеками для парсинга выступают:
1. requests - делает запрос на сайт и получает данные
2. bs4 (BeautifulSoup) - парсит результат запроса requests

Код 
import requests
from bs4 import BeautifulSoup
import polars as pl
import re
import xlsxwriter
import time

df = pl.DataFrame({             # 1
    "Вакансия": pl.Series([], dtype=pl.Utf8),
    "Зарплата": pl.Series([], dtype=pl.Int64),
    "Keyword": pl.Series([], dtype=pl.Utf8),
    "Tags": pl.Series([], dtype=pl.Utf8),
    "URL": pl.Series([], dtype=pl.Utf8)
})

#^2
start_url = "https://hh.ru/search/vacancy?experience=between1And3&order_by=publication_time&ored_clusters=true&schedule=remote&text=sql&search_period=7"

url = start_url                 # 3

headers = {                     # 4
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36",
    "Accept-Language": "en-US,en;q=0.9",
    "Accept-Encoding": "gzip, deflate, br",
    "Connection": "keep-alive",
    "Upgrade-Insecure-Requests": "1"
}

keywords = ["BPMN", "Jira"]     # 5
  1. Создаем пустую таблицу, в эту таблицу будет добавляться найденная информация о вакансиях
  2. Стартовая страница запроса, эта строка учитывает указанные фильтры на hh, то есть, можно перейти на hh, указать критерии поиска, выбрать фильтры, а после скопировать ссылку и вставить в скрипт
  3. Эта ссылка служит основной для перехода, что бы перейти на вторую страницу нужно к ссылке добавить page=номер страницы, я использую код url=f’{start_url}&page={p}’ который имеет константу страницы start_url, и конкатинацию в зависимости от итерации цикла
  4. Эти настройки нужны при запросе requests.get(), для имитации, что запрос на страницу делает пользователь, когда сайт запрашивает данные, мы ему предоставляем данные из этого списка
  5. Ключевые слова, которые скрипт будет искать на страницах вакансий

Проверка полученной страницы (вакансия)

При запросе страницы бывает приходит страница не полная, у которой нет части данных. Эта функция проверяет наличие данных, если они не полные то делает запрос повторно

Код 
def check_correctly(url):
     for i in range(9):         # ^1
          response = requests.get(url, headers=headers) # 2
          soup = BeautifulSoup(response.text, 'html.parser') # 3

          header = soup.find('div', {'class': 'vacancy-title'}) # 4
          if header:                                            # 5
               visit_and_check(url, soup, response)             # 6
               break
  1. Если данные приходят не полные, то повторяем запрос
  2. Сам запрос на получение данных с сайта, url - ссылка данные которой запрашиваем, headers - это имитация пользователя которая создавалась ранее
  3. Парсим полученный объект (теперь в этой переменной лежит “грубо говоря” html страница), которую можно парсить. Преимущество в том, что при тестах не делается запрос на сайт
  4. Ищем div class с названием vacancy-title
  5. Проверяем переменную header,(для чего ? читай описание заголовка), если мы получили полную страницу
  6. вызываем функцию и передаем url - ссылку, soup - локальную html, response - для поиска слов по всей странице

Сканирование вакансии

Эта функция извлекает нужные данные страницы

Код 
def visit_and_check(url, soup, response):
  global df                     # 1

  fkeys = []                    # 2

  for keyword in keywords:      # 3
      if keyword in response.text: # 4
        fkeys.append(keyword)      # 5

  # Находим теги (навыки) в нижней части страницы
  skill_elements = soup.find_all('li', {'data-qa': 'skills-element'}) # 6
  skills = [li.find('div', class_=re.compile(r'magritte-tag__label')).text for li in skill_elements] # 6.1

  header = soup.find('div', {'class': 'vacancy-title'}) # 7
  title = header.find('h1').text                        # 8

  salary_str = header.find('span', {'data-qa': 'vacancy-salary-compensation-type-net'}) # 9
  if salary_str:                # 9.1
    # Найти первое число в строке с помощью регулярного выражения
      match = re.search(r'\d+', salary_str.text.replace('\xa0', '')) # 9.2
      if match:
        salary = int(match.group()) # 9.3
      else:
        salary = None           # 9.4
  else:
    salary = None               # 9.4

  df = df.vstack(pl.DataFrame({ # 10
    "Вакансия" : [title],
    "Зарплата" : [salary],
    "Keyword": [', '.join(fkeys)],
    "Tags": [', '.join(skills)],
    "URL": [url]}))
  1. Эта строка нужна для из за пространства имен, дело в том что данная функция вызывается для каждой вакансии и результат полученных данных нужно записать в таблицу. (после выхода из функции все объекты созданные в ней удаляются), по этому таблица создается вне функции, глобально, и что бы получить доступ к глобальной таблице нужна эта строка
  2. В этот список будут помещены слова найденные на странице (список слов указан в переменной keywords)
  3. Собственно сам поиск этих ключей указанных в keywords, берется каждый ключ по очереди
  4. И производиться поиск ключа (слова) по объекту response.text который вернул запрос к сайту
  5. Если ключ найден на странице он добавляется в пустой список fkeys (который создали ранее)
  6. Ищем нужные данные в локальном html который спарсили, в данном случае, это теги в нижней части страницы
    1. На прошлом шаге мы получаем кусок html кода с нужными тегами, данная строка извлекает текст тега и помещает в список
  7. Извлекаем header
  8. Получаем название вакансии
  9. Получаем ставку (указанную зарплату)` Примечание: есть вакансии на которых указанной зарплаты нет, а бывают случаи когда указан диапазон от…до, такие моменты обрабатываю ниже
    1. проверяем, указана зарплата
    2. регулярным выражением ищем только число (убираем символы)
    3. преобразовываем в число
    4. если зарплата не указано, помечаем переменную как Nane
  10. полученные данные записываем в таблицу

Переход по страницам и поиск ссылок

Этот цикл сканирует главные страницы (на которых находятся вакансии), в зависимости от итерации переходит по страницам 2,3,4… на самом сайте

Код 
for p in range(9):              # 1
    if p > 0:
        url = f'{start_url}&page={p}' # 2
        print(f"--- Page {p} --- --- ---")

    # Поиск всех ссылок внутри тегов h2
    for i in range(9):          # 3
        time.sleep(3)           # 4
        print(f"--- --- --- Iteration {i}")
        response = requests.get(url, headers=headers) # 5
        soup = BeautifulSoup(response.text, 'html.parser') # 6
        if soup.find_all('h2'):                            # 7
            for h2 in soup.find_all('h2'):                 # 7.1
                for a in h2.find_all('a', href=True):      # 7.2
                    absolute_url = requests.compat.urljoin(url, a['href']) # 7.3
                    check_correctly(absolute_url) # 8
            break                                 # 9
  1. Указываем кол-во страниц (главных) по на которых будет смотреть вакансии
  2. Если это не первая итерация цикла, то к ссылке добавляем page и цифру страницы (цифру итерации цикла)
  3. Бывает hh.ru возвращает не полную страницу, в этом цикле проверяем корректность полученной страницы, если она не полная повторяем запрос вплоть до получения полной страницы или в плоть до числа в функции range()
  4. Перед каждым новым запросом немного ждем, что бы не DDOSить сайт, все таки цель получить данные, имитируем простой переход
  5. Сам запрос на получение данных с сайта, url - ссылка данные которой запрашиваем, headers - это имитация пользователя которая создавалась ранее
  6. Парсим полученный объект (теперь в этой переменной лежит “грубо говоря” html страница), которую можно парсить. Преимущество в том, что при тестах не делается запрос на сайт
  7. Проверяем полученную страницу, если она не полная то делаем запрос еще раз (переходим на новую итерацию)
  8. Если страница полная, извлекаем h2 (заголовок вакансии)
  9. В этом заголовке ищем ссылку на вакансию
  10. Из этой ссылки делаем абсолютный путь (полную ссылку на вакансию)
  11. Передаем ссылку на вакансию в переменную проверки
  12. На данном этапе то как мы попали в if soup.find all(‘h2’), значит мы получили полную ссылку на страницу, делать новый запрос по той же ссылке не нужно
